## Overview

FakeShield is a complete machine learning pipeline that:
1. Ingests the Kaggle fake-and-real news dataset (~44,000 articles)
2. Cleans and normalises text using NLTK (stopwords, lemmatization, URL removal)
3. Vectorises articles using **TF-IDF** (50,000 features, bigrams)
4. Trains two classifiers: **Logistic Regression** and **Multinomial Naive Bayes**
5. Evaluates with Accuracy, F1 (macro + weighted), Precision, Recall, ROC-AUC
6. Serves predictions via a polished **Streamlit** web application

| Feature | Description |
|---|---|
| NLTK Text Cleaning | Lowercase, URL/HTML removal, stopwords, lemmatization |
| TF-IDF Vectorization | 50K features, unigrams + bigrams, sublinear TF scaling |
| Dual Models | Logistic Regression + Multinomial Naive Bayes |
| Rich Evaluation | F1, accuracy, confusion matrix heatmaps, ROC curves |
| Streamlit App | Dark-themed UI with confidence gauge, probability bars |
| Word Explanations | Top LR coefficient features for each prediction |

## 📦 Dataset Setup

The project uses the **Fake and Real News Dataset** by Clément Bisaillon on Kaggle.

### Option A — Kaggle API (automatic)
```bash
# Install Kaggle CLI and set up your API token
pip install kaggle
# Place kaggle.json in C:\Users\<you>\.kaggle\
python data/download_data.py
```

### Option B — Manual download
1. Go to: https://www.kaggle.com/datasets/clmentbisaillon/fake-and-real-news-dataset
2. Download and extract the ZIP
3. Place **`Fake.csv`** and **`True.csv`** inside the `data/` folder

## 🚀 Usage

### Train Models (required first)
```bash
python train.py
```
This runs the complete 6-step pipeline:
```
[1/6] Loading dataset
[2/6] Preprocessing text
[3/6] Splitting data (80/20 stratified)
[4/6] Vectorising with TF-IDF
[5/6] Training models
[6/6] Evaluating models → saves plots + metrics
```

### Launch the Web App
```bash
streamlit run app.py
```
Open your browser at http://localhost:8501

### Explore the Jupyter Notebook
```bash
cd notebooks
jupyter notebook news_classification.ipynb
```

## 📝 Notes

- The dataset is nearly balanced (~23,500 fake / ~21,400 real), so accuracy is a fair metric alongside F1.
- The `cleaned_content` column combines article **title + body** for richer features.
- Logistic Regression coefficients are used directly for word-level explanations — positive weights indicate REAL, negative weights indicate FAKE.
